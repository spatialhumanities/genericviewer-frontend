pundit.contextualMenu Class

Extends pundit.baseComponent
Defined in: src/ContextualMenu.js:1
Module: pundit

Provides methods and callbacks to open a contextual menu with customizable options and actions.
The actions are organized by "types", various components can activate the contextual menu for a certain type, and all of the actions subscribed with that type will be shown.
Each action can have its own function which decides wether to show it or not, and its own onClick callback to be executed.
When the contextual menu is shown, an unique identifier or a complex data structure must be passed in, allowing each action to run their showIf() function, and eventually the onclick().

addAction

(
  • action
)

Defined in src/ContextualMenu.js:89

Adds an action for the given types.
Example of the action object: 

{
....type: ['myUniqueType'],
....name: 'removeMyItem',
....label: 'Remove this DOM item',
....showIf: function(id) { 
........return dojo.query('#'+id).length !== 0;
....},
.....onclick: function(id) {
........dojo.destroy(id);
....}
}

This action will destroy a DOM node with the given id. The menu option will be shown if an element with the given id exists.

Parameters:

  • action Object
    • type Array of strings

      : List of types to subscribe this action to.
      The special type "__all" means that this action must be shown disregarding the type passed to the show() function.

    • name

      (string): Unique name of this action, used for HTML classes, so no spaces or strange characters are allowed. To be safe just use camelcase letters.

    • label String

      : Label to be displayed for this menu action.

    • showIf(data) Function

      : function which decides wether or not show this action in the menu. The data parameter passed in is the same parameter passed when showing the menu. Usually an URI or an identifier useful to the subscribing component to make its job.

    • onclick(data) Function

      : Function to be called when the users clicks on this action. The data parameter passed to the callback is an unique identifier of the item we are showing the menu for, or an equivalent complex structure. Usually just an identifier or an URI.
      If the onclick() function returns true, the contextual menu will be hidden.

Inherited from pundit.baseComponent: src/BaseComponent.js:71

Creates one or more callbacks for the component. For each 'name' passed as parameter two methods will be created:
onName(f) (first letter is automatically capitalized): used by other components to subscribe a function to be called when the event hits. Optional parameters.
fireOnName(data) (first letter is automatically capitalized): fires the event calling all of the subscribed callbacks passing data as parameter. This function must be called by the component when needed.

Parameters:

  • names String or array of strings

    Names of the callbacks to be created.

Shows the contextual menu for the given type and identifier, at the given x,y coordinates.
Each component can open a contextual menu for a given type, passing in an identifier which will be used by showIf() and onclick() functions to do their job.

Parameters:

  • x Number

    x coordinate where to show the contextual menu

  • y Number

    y coordinate where to show the contextual menu

  • url String

    unique identifier to identify the clicked resource

  • type String

    type of actions to show
